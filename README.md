**git-intro

This repository provides an introduction to Git and version control concepts through practical examples and exercises. It is designed to help beginners become familiar with essential Git workflows and file management in a collaborative coding environment.

Files included
.gitignore: Specifies which files or file patterns should be ignored by Git (e.g., log files).

README.md: This project overview and instructions.

greeting.txt: Example text demonstrating basic file creation and adding content.

improvement.txt: Example file for practicing feature improvements and branch merging.

message.txt: Example file for practicing message updates and merging branches.

Learning objectives
Understand key Git commands and concepts: adding, committing, pushing, branching, and merging.

Practice file management and collaboration in a Git repository.

Learn how to use a .gitignore file to exclude unnecessary files from version control.

Get comfortable with making improvements, switching branches, and merging changes.

Getting started
Clone the repository:

bash
git clone https://github.com/Loic2888/git-intro.git
Explore the files and read the content of each text file.

Practice basic Git commands:

Add new files: git add <filename>

Commit changes: git commit -m "Your message"

Create and switch branches: git checkout -b <branchname>

Merge branches: git merge <branchname>

Push changes to GitHub: git push

Experiment by making changes and following best practices for commit messages and repository management.
